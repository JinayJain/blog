Why You Should Learn Competitive Programming

Jinay Jain

Competitive programming has consistently increased in popularity over the years, partly due to the fact that more and more companies use technical coding interviews to screen their candidates. But the fruits of algorithmic thinking extend beyond simply passing the Google coding interview: they give you a renewed perspective on every for loop and if statement you'll ever write.

Competitive programming is an art form that few have truly mastered. Like chess, every new level in your competitive career brings you more difficult challenges, tougher competition, and more rewarding work. The solutions are often very simple, but the intuition behind them stems far beyond what few lines of code you see. Here's a recent solution I had for a Codeforces round.

What if I wanted to test some inline command like cat cats.txt in my code.

#include <bits/stdc++.h>

using namespace std;

int main()
{

    int T;
    cin >> T;

    for (int t = 0; t < T; t++)
    {
        int n;
        cin >> n;
        vector<int> arr(n);
        vector<int> pos(n + 1);

        for (int i = 0; i < n; i++)
        {
            cin >> arr[i];
            pos[arr[i]] = i;
        }

        // for (int i = 1; i <= n; i++)
        // {
        //     printf("x: %d pos[x]: %d", i, pos[i]);
        //     cout << endl;
        // }

        int curr = 1;

        bool possible = true;

        while (curr < n)
        {
            int p = pos[curr];

            // cout << curr << endl;

            for (int i = p; i < n; i++)
            {
                if (arr[i] < curr)
                {
                    break;
                }
                if (arr[i] != curr)
                {
                    possible = false;
                    break;
                }
                curr++;
            }

            if (!possible)
                break;
        }

        cout << (possible ? "Yes" : "No") << endl;
    }

    return 0;
}
